HELP FILE FOR DOING STRUCTURAL SIMILARITY SEARCHES

  • Exact Search: An exact structure search finds molecules that are equal (in size) to the query structure. (No additional fragments or heavy atoms are allowed.)
  • Similarity Search: Similarity is only used in database searches, and its similarity concept is based on hashed binary chemical fingerprints with Tanimoto metrics. (For a more detailed description, see the Developers Guide.) For a more sophisticated approach of similarity, we provide the Screen package.
  • Perfect Search: Perfect search is mainly used before database inserts to check whether the given molecule already appears in the database or not. All molecular features need to be equal here, eg. non-stereo query will only match non-stereo target, etc.
  • Substructure Search: Substructure search, that is, whether one molecular structure contains the other one as a substructure or not
  • Superstructure Search: Superstructure search is the opposite of substructure search: It searches for those target molecules which can be found in the given superstructure query. (In this case the roles of the query and target molecules are simply exchanged, so query properties should be specified on the target!)
